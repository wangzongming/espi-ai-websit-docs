この记事では、制御されるリモートデバイスをEnd-endと呼び出します。年末の終わり。

さて、オープンプラットフォームターミナルは `8266系列开发板` をサポートしています。(さらに多くのハードウェアが開発中です...)

安価な `esp01s`、`esp01` などを使うことをお勧めします。スイッチの機会は、`esp01s relay` リレー発行のこの开発ボードを强くお勧めします。

以下は以下です。

＃＃１．年末の終わり。

1. 开発ボードをコンピュータに开続します (`esp01s` にはバーナーが必要です)
2.オープンプラットフォームを開設、スーパーボディページを開設します
3. 左上隅の左上隅 [**残念の书き込み**]。
4. デバイスを开続し、书き込み用の法保留の下にある `超级开关` 恐れ入りますを選択します。
5. 书き込み後、ウィンドウを閉じて、ログボタンをクリックしてそのままポートログを开きます（ログを意味するためのものです）。
6. `ESP-AI-endpoint` `192.168.4.1`
7. ネットワークが正常に構成されると、`开放平台-> 设备列表 -> 终端设备` にもう 1 台のデバイスが存在することがわかります。
8. 上記は当てはまりません。


## 2. スーパースイッチ配線図
以下の回路と5V電圧の解析。電源 コードのプラス(赤) ワイヤーを切りますし、両エンドを下の表の `COM` と `NO` にそれぞれ違いし込みます。

`220v`のような電源制御やする場合は、下表のリチウム電池の圧力ドロッパーを省略し、圧力ドロッパーを追加します（**危険な動作です）。専属家族以外で運営されています**)。

| ESP01S リレー |電源コード (プラスとマイナスのワイヤーが発行しています) | リチウム電池 (3.7v) |
| ------------ | ------------------------ | -------------- |
| +            |                          | +              |
| -            |                          | -              |
|コム |ポジティブ | - |
|いいえ |ポジティブ | - |

## 4. リモートデバイスの制御が進行中 (Nodejs コード)
コードを分割記載する場合は、`target_device_id`、`api_key`をコマンド憲法に追加必要があります。

```js
const config = {
    gen_client_config: async (){
        return {
            intention: [
                { 
                    key: ["开灯", "打开灯"],  
                    message: "好的",  
                    instruct: "__io_high__", 
                    pin: 0, // 如果使用的是 esp01s relay 的话这里写0就行，因为它用IO0来控制继电器
 
                    // 目标设备ID: 开放平台-> 设备列表 -> 终端设备 打开后就能看到一个设备ID
                    target_device_id:"xxx",
                    // 超体 api_key
                    api_key:"xxx",
                },
                
                { 
                    key: ["关灯", "关闭灯"],  
                    message: "好的" 
                    instruct: "__io_low__", 
                    pin: 0, // 如果使用的是 esp01s relay 的话这里写0就行，因为它用IO0来控制继电器

                    // 目标设备ID
                    target_device_id:"xxx",
                    // 超体 api_key
                    api_key:"xxx",
                }
            ]
        }
    }
}
```
