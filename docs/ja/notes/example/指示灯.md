`ESP-AI` は、表示灯 (`ws2812`) `IO18` しないでください。


ビジネスロジックでも意思決定ーを使用する必要がある場合は、`esp_ai_pixels` を使用して操作できます。`esp_ai_pixels` は `Adafruit_NeoPixel` の例です。詳しい内容はこちら。一般的な色の使い方と消灯制御方法を以下に示します。。

```c
// 将灯改为红色
esp_ai_pixels.setPixelColor(0, esp_ai_pixels.Color(238, 39, 70));
esp_ai_pixels.setBrightness(100);  // 亮度设置
esp_ai_pixels.show();

// 熄灭灯
esp_ai_pixels.clear();
esp_ai_pixels.show(); 
```
## 照明の説明
 
|ライトステータス |色 |説明 |タイプ |
| -------- | ------ | ------------------------------------------------------- | -------- |
|オフ| - |スタンバイステータス、ウェイクワードアップをHUび出してウェイクアップ |正常なステータス |
|长にオン |ザボンイエロー |フォトネットワークはデバイスのホットスポットを構成するために継続する必要があります |正常な状態 |
|指摘する |楽天 |。機会を受け取り、機会の扉を開ける。正常な状態 |
|长にオン |美しいバタフライグリーン |写真AIが音声認識ステップに入る |通常状態 |
|いつも明るい |チュンメイホン | AIが答えステップに入る |通常状態 |
|ゆっくり指摘 |チュンメイホン|秘密のキーはバインドがないため、ネットワークもう一度構造を作る必要がある。正常な状態 |
|常に点灯 | ライトレッド |

##勝手の指示ピンを変长する

只需要在 `#include <esp-ai.h>` 代码前定义一下变量即可。
```c
// 更改指示灯引脚方式，默认 IO18
#ifndef LED_BUILTIN
#define LED_BUILTIN 10 
#endif

#include <esp-ai.h>
// ...
```
