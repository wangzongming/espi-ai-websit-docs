## 会話速度の最適化

回答の対応回答が遅いと敏感になった場合は、以下の項目をよくご確認ください。
1. `LLM`, `IAT`, `TTS` の 3 つのサービスの対応 応答速度は、コンソールが情報を出力するのにかかる時間にベースづいて計算できます。
2. `auth` `await` `await` を設定します。`auth`。キャッシュされた情報は、インターフェイスにリクエストを行う代わりに `auth` を使用する必要があります。毎回`mysql`。
3. `指令配置` は多すぎますか?
4. コマンドラインは非同期であり、応答時間が長くなります。。(つまり、コマンド設定項目`key`でリクエストが発行され、リクエストサービスの応答時間が長いです)
5. 物理サーバーの帯区综合が少なすぎますか (6mb 不満の折、24k はとても遅いです)
6. 音声認識の無音時間の設定はどのくらいですか? 1.5秒。。

**オファー:**
回答時間:IAT応答時間 + 音声認識時間 + LLM応答時間 + TTS応答時間 + コマンド解析時間 + サーバー遅延

##オンライン開発提案
ESP-AIをサービスとしてINDEPENDENTとしてデプロイし、`PM2`などのツールを使ってプロジェクトを実行し、マルチスレッドサービスを実装することをお勧めします。

また、負荷分散を実現するには、`Nginx` サービスを使用することをお勧めします。

ただし、これにより各デバイスが異なるサービスに分散されることに注意してください。デバイスはコマンドを再起動するなどメッセージを送信するなど、サーバーがデバイスを制御するときは、デバイススサービスにアクセスするたびに記録を成さない制限、各ESP-AIサービ スを制御します。。
