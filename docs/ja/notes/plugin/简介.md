プラグインは3つのタイプに分類されます。 `TTS` | `IAT`。各プラグインは、フレームワークによって提供される様々な仮パラメータを受け取って、パラメータしてフレームワークと通信します。

のプラグインの種類であっても、基本は四角に同じです。基本的なロジックは取得した後、オープナーがカスタムサービスデータを、そしてESP-AIフレームワークによって提供されるメソッドを使って、それをESP-AIに多すというものです。。


## 必要な能力

多くの人は、プラグインの成りは多しく、ハードウェアやサーになるのではないかと心配しています。実记、それは想像されているほど狠しくありません。プラグインは、ハードウェアの一部とその一部。**开者は、自分のサービスのデータをESP-AIに多すメソッドのみを考慮する必要があります。 **

`Nodejs` をしてプラグインを使用して「します」を作成します。各マニュアルと完全なマニュアルの説明が提供されます。。説明: ファイルを `Chat GPT`。ただし、开者は特定の `Nodejs` プログラミング能力も必要です。

`ESP-AI` は、より多くの语でプラグインを开発できるように `SDK` パッケージを提供しています。まず、`Python` `ESP-AI` プラグインを提供する `SDK` について調整していきます。気長にお台ちください...

## 規制は遵守しなければなりません

1. LLM プラグイン名は、`esp-ai-plugin-llm-xxx` の形式である必要があります。
2. TTSプラグイン名は、`esp-ai-plugin-tts-xxx`の形式である必要があります。
3. IAT プラグイン名は、`esp-ai-plugin-iat-xxx` の形式である必要があります。
4. プラグインを外に始めるには、時のキーワードをプラグインディレクトリの `package.json` に必要な情報を追加する必要があります。
``` json
{
    "keywords": ["esp-ai-plugin", "esp-ai-plugin-llm"] 
}
```

## スタートガイド
LLMの基本構造 | TTS | IAT。

``` javascript
const config = {
     plugins: [
        {
            name: 'esp-ai-plugin-llm-test', // 插件名称  
            type: "LLM",// 插件类型 LLM | TTS | IAT
            main(arg){
                // 插件逻辑
            }
        }
     ]
}
```


次に、コード標準の機会に、プラグイン配置配列にダイレクトブックを入れることはできません。したがって、プラグインを成す通常の手は時の正しいです。
1. プラグイン解除 `esp-ai-plugin-llm-test` を成します。
2. `package.json` します
``` json
{
  "name": "esp-ai-plugin-llm-example",
  "keywords": ["esp-ai-plugin", "esp-ai-plugin-llm"],
  "version": "0.0.2",
  "description": "ESP-AI LLM Plugin Example | ESP-AI LLM 插件开发案例",
  "main": "index.js",
}
```
3. `index.js` オリジナル作品に追加
``` javascript
module.exports = {
    name: 'esp-ai-plugin-llm-test', // 插件名称  
    type: "LLM",// 插件类型 LLM | TTS | IAT
    main(arg){
        // 插件逻辑
    }
}
```
4. `npm` に開きます。プライベートプラグインの機会に、この心如はされます。
   
２回目は公開です。。はい。
```
npm publish
```

5. プラグインコレクション
プラグインが `npm` に开された後、プラグインをオープンプラットフォームに自分で信すると、プラグインが官方にグループに受け入れられ、より多くの人が见ることができるようになります。
